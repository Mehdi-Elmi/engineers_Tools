# قوانین تعامل محیط طراحی

این فایل قانون‌های تعامل فعلی برای محیط `Engineering Design Tools` را ثبت می‌کند.

## مسیر فعال

```text
modules/mechanics_dynamics_statics/workspace.py
modules/mechanics_dynamics_statics/interaction_fixes.py
modules/mechanics_dynamics_statics/window_resize_fixes.py
```

`interaction_fixes.py` فعلاً اصلاحات کم‌ریسک interaction را روی محیط فعال نصب می‌کند تا فایل‌های بزرگ و پایدار پروژه بی‌دلیل بازنویسی نشوند.

`window_resize_fixes.py` رفتار resize پنجره frameless را نصب می‌کند.

## قانون گزارش کار

از این مرحله به بعد، هر تغییر باید با این ترتیب انجام شود:

1. اعلام شود دقیقاً چه چیزی بررسی می‌شود.
2. اعلام شود دقیقاً کدام فایل یا بخش تغییر می‌کند.
3. بعد از تغییر، گفته شود کاربر باید کدام بخش برنامه را تست کند.

## قانون Resize پنجره اصلی

- resize پنجره یعنی تغییر viewport و فضای دیده‌شدن برنامه، نه تغییر اندازه واقعی صفحه مهندسی.
- fullscreen، restore، minimize یا کوچک/بزرگ کردن پنجره نباید ابعاد مهندسی صفحه را عوض کند.
- اگر صفحه در حالت fullscreen بزرگ‌تر دیده می‌شود، این فقط scale نمایش/fit view است؛ مقدار واقعی X/Y، واحد، grid و object نباید عوض شوند.
- محورهای X و Y باید قانون مثبت/منفی ثابت داشته باشند و با تغییر اندازه پنجره جابه‌جا یا وارونه نشوند.
- برای پنجره frameless، hover روی لبه‌ها و گوشه‌ها باید قبل از کلیک cursor resize را نشان دهد.
- cursor resize باید با رنگ و کنتراست مناسب روی سرمه‌ای و پس‌زمینه‌های تیره دیده شود.
- گوشه بالا-چپ و پایین-راست باید cursor قطری `\` داشته باشند.
- گوشه بالا-راست و پایین-چپ باید cursor قطری `/` داشته باشند.
- drag از لبه چپ فقط left پنجره را تغییر می‌دهد؛ right ثابت می‌ماند.
- drag از لبه راست فقط right پنجره را تغییر می‌دهد؛ left ثابت می‌ماند.
- drag از بالا فقط top پنجره را تغییر می‌دهد؛ bottom ثابت می‌ماند.
- drag از پایین فقط bottom پنجره را تغییر می‌دهد؛ top ثابت می‌ماند.

## قانون اندازه واقعی Workspace

- اندازه واقعی صفحه اصلی workspace در این مرحله `400 x 220 mm` است.
- در مکالمه ممکن است به صورت `220 x 400` گفته شود، اما قانون اجرایی فعلی در کد به شکل `(400.0, 220.0)` ثبت می‌شود: عرض 400، ارتفاع 220.
- resize پنجره نباید این مقدار را به `430 x 250` یا عددهای مشابه تبدیل کند.
- تغییر پنجره فقط viewport، layout و نمایش را تغییر می‌دهد.

## قانون Resize آبجکت

- صفحه یا canvas نباید به خاطر resize آبجکت تغییر اندازه واقعی بدهد.
- resize آبجکت باید با anchor درست انجام شود؛ یعنی وقتی کاربر از لبه چپ می‌کشد، لبه راست باید مرجع بماند و برعکس.
- گوشه‌ها و بالا/پایین باید hit-test بزرگ‌تر و قابل گرفتن داشته باشند.
- حداقل اندازه آبجکت باید رعایت شود، اما clamp نباید باعث جابه‌جایی نادرست anchor شود.

## قانون Cursor و Indicator

- از cursorهای خام Qt مثل `SizeHorCursor`, `SizeVerCursor`, `SizeFDiagCursor` به عنوان ظاهر نهایی استفاده نشود.
- cursorهای move، resize و ruler guide باید از الگوی گرافیکی پروژه پیروی کنند.
- cursor دوران نباید فلش دوران باشد.
- حالت عادی روی handle دوران باید hand/open-hand باشد.
- وقتی کاربر با کلیک چپ دوران را شروع می‌کند، cursor باید closed-hand شود.
- این قانون بعداً برای ابزارهای line، snap، rotate، magnet و constraint هم باید مبنا باشد.

## قانون منوی راست‌کلیک

- منوی راست‌کلیک خطکش، guide، canvas و آبجکت نباید باکس خام و نوک‌تیز سیستم باشد.
- هر context menu باید با الگوی گرد و هماهنگ پروژه نمایش داده شود.
- اگر جایی QMenu خام استفاده شده باشد، باید یا با ProjectMenuDialog جایگزین شود یا mask و style واقعی بگیرد.

## قانون Ruler و Zoom

- ruler باید با zoom فعلی canvas هماهنگ باشد.
- وقتی zoom تغییر می‌کند، فاصله tickهای ruler باید با همان scale بصری canvas هماهنگ شود.
- origin خطکش باید به همان مختصات بصری grid اشاره کند و از canvas جداگانه رفتار نکند.
- فعال کردن صفر-صفر خطکش از گوشه نباید باعث تغییر اندازه صفحه شود.

## قانون نمایش زاویه

- هنگام rotation، مقدار زاویه باید فقط نزدیک handle دوران نمایش داده شود.
- زاویه نباید همزمان در چند جای UI تکرار شود مگر کاربر صریحاً بخواهد.
- زاویه نمایش داده‌شده باید از مبنای صفر محاسبه شود.
- اعداد اعشاری باید حفظ شوند؛ مثلاً `10.25 deg` باید واقعاً 10.25 درجه اعمال و نمایش داده شود.
- واحد زاویه در UI فعلی به صورت `deg` نمایش داده می‌شود تا با کد و کنترل‌های فعلی سازگار بماند.

## قانون کنترل Degree

- کنترل degree باید ورودی دستی اعشاری داشته باشد.
- فلش‌های بالا/پایین کنترل degree باید از همان الگوی فلش‌های کنترل پایین صفحه در StatusBar/Zoom پیروی کند.
- هیچ کنترل degree نباید با دو مدل فلش یا دو زبان بصری مختلف ساخته شود.

## قانون F8

`F8` میانبر پیش‌فرض Orthogonal Snap است.

رفتار فعلی:

- اگر آبجکت انتخاب‌شده rotation داشته باشد، با `F8` به نزدیک‌ترین محور ۰، ۹۰، ۱۸۰ یا ۲۷۰ درجه snap می‌شود.
- `F8` باید هم در سطح پنجره و هم زمانی که focus روی Canvas است کار کند.
- اگر آبجکت انتخاب نشده باشد، فقط StatusBar پیام راهنما نشان می‌دهد.
- این رفتار بعداً باید برای ابزارهای رسم مثل Line و Vector هم استفاده شود تا خط در نزدیکی محورهای XY افقی یا عمودی شود.

## قانون Properties

Properties باید `Shortcut Keys` و `F8: Orthogonal Snap` را به عنوان گزینه پیش‌فرض محیط طراحی نشان دهد.
ولی مقدار زاویه نباید همزمان در Properties و کنار handle تکرار شود؛ نمایش اصلی زاویه کنار handle دوران است.

## نکته معماری

این فایل جایگزین مستند اصلی `docs/window_pattern.md` نیست. اگر این قانون‌ها نهایی شدند، باید بعداً از حالت patch runtime خارج شوند و به کد اصلی `start_bar.py`، `module_window.py` و `workspace.py` منتقل شوند.
